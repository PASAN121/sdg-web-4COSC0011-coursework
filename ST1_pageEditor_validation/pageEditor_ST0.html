Home page

Technical Description

The Home Page was developed using semantic HTML5 elements such as <header>, <main>, <section>, and <footer> to ensure a clear and meaningful structure. The page layout is divided into a hero section, a card-based information section, and an interactive gallery..

CSS Grid was used to create responsive layouts, particularly with grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)), allowing content cards to automatically adjust based on screen size. The hero section uses a background image with background-size: cover and background-position: center to maintain visual consistency across different devices.

The gallery section includes hover effects implemented using CSS transitions and transforms such as transform: scale() and translateY(). These effects enhance interactivity and user engagement. The overall design focuses on simplicity, readability, and a modern user interface.

Accessibility

Accessibility was considered by using descriptive alt attributes for all images to support screen readers. Semantic HTML elements improve navigation for assistive technologies. Text contrast was carefully selected to ensure readability, especially in the hero section where white text is placed over a dark overlay.

Gallery Page

Technical Description

Explain the HTML and CSS techniques used to implement this page. Justify the logic and purpose of the key tags and styling choices focusing on functionality and design without copying large code sections. Explain the purpose of key HTML tags and CSS properties. Specify key HTML tags and CSS properties used to create structure and styling.

Each gallery item is placed inside a div with the class .cell, containing an image and a label. CSS properties such as object-fit: cover and aspect-ratio maintain consistent image dimensions. Hover effects are created using transform: scale() and box-shadow to enhance interactivity.

JavaScript is used to implement a dynamic modal system. When a gallery image is clicked, event listeners (addEventListener()) capture the interaction and update the modal content. The selected image source, title, and description are dynamically inserted into elements such as modalImage, modalTitle, and modalDescription.

The modalBody element acts as the main container for the popup content. It is dynamically updated using JavaScript to change styles such as background color, font, and text color. Additional controls (dropdown menus) are created using createElement() and inserted into the DOM using appendChild(), allowing users to customize the modal appearance in real-time.

The modal visibility is controlled by toggling the .active class using JavaScript. Smooth animations are achieved using CSS transitions and transforms, creating a modern and interactive user experience without reloading the page.

Accessibility

Accessibility was considered by including descriptive alt attributes for all images, ensuring that users with screen readers can understand the content. Semantic headings (<h2>) are used to clearly structure different sections such as Wildlife Protection, Deforestation, and Take Action.

The modal design ensures readability with proper contrast between text and background. Interactive elements such as buttons and dropdowns are clearly labeled, improving usability. The responsive grid layout also ensures accessibility across different devices including mobile and tablets.

Content Page

Technical Description

This page is implemented using semantic HTML5 elements and CSS styling to create a structured, responsive, and visually appealing layout. Key HTML tags include <header> for site navigation, <main> to contain primary content, <section> and <div> for organizing content blocks, <h2> and <h3> for headings, <p> for paragraphs, <ul> and <li> for lists, and <img> for images with descriptive alt text.

CSS is used to create layout and style. Flexbox (display: flex;, justify-content, align-items) is applied for card and list layouts to allow flexible alignment and spacing. Images use object-fit: cover; and border-radius for proper scaling and rounded corners, while box-shadow adds depth. Interactive effects like transition and :hover provide subtle animation for user engagement. Classes like .fact-box highlight important information with background color, border, and padding. The sticky "Go to Top" button (position: sticky;) improves navigation across long content.

Overall, the combination of semantic HTML and targeted CSS ensures that the page is structured, visually organized, and responsive while supporting user interaction and readability.

Accessibility

Accessibility is addressed by using semantic tags (<header>, <main>, <section>) which help screen readers understand content structure. Images include descriptive alt attributes for users with visual impairments. Text contrast and line-height are chosen for readability. Navigation links are keyboard accessible and focusable. Interactive elements, like buttons and "Go to Top" links, are properly labeled. Using list semantics for menus and content improves clarity, while consistent headings and content order maintain logical reading flow for assistive technologies.

Challenges and Lessons Learned

During this project, I initially faced several challenges, especially in understanding how to work with JavaScript objects and arrays effectively. At first, managing data structures and accessing nested values was confusing, but I improved my understanding by following a LinkedIn Learning JavaScript course. This helped me apply concepts more confidently in practical scenarios.

I also struggled with implementing advanced CSS features such as overlays, smooth hover effects, and transitions. Creating visually appealing and interactive elements required a deeper understanding of timing, positioning, and styling techniques. Through practice, I was able to improve these aspects and achieve smoother user interactions.

Another challenge was implementing smooth scrolling behavior across the website. Initially, it did not work as expected, but by researching and testing different approaches, I was able to refine it and create a better user experience.

Additionally, while working with CSS, I noticed that I had to repeatedly use the same colors and styles across different sections. This made the code harder to manage and maintain. To solve this, I learned how to use CSS root variables (:root), which allowed me to define global styles such as colors and reuse them efficiently throughout the project.

Overall, these challenges helped me learn and grow my skills in JavaScript and CSS, and I was able to overcome them through consistent practice and learning.

Compliance with Janet Regulations

While developing my web pages, I ensured that the content and implementation adhered to Janet regulations and best practices. The pages are intended for educational purposes, and I avoided any content that could be deemed inappropriate or illegal.

I followed GDPR and UK Data Protection Act principles by not collecting any personal data from users. Any external content, such as images and fonts, were properly attributed and used under copyright rules, in line with ethical and legal web development practices.

For security, all pages are designed to avoid malicious code, and any sensitive information is protected. Although this project is for learning purposes and does not involve real user data, I am aware of Janet CSIRT procedures and would report any potential security issues if they arose.

Accessibility and inclusivity were also considered. I used semantic HTML elements such as <header>, <nav>, and <section> to structure content properly. Images include alt attributes, colors were chosen for high contrast, and interactive elements are navigable via keyboard to comply with WCAG 2.1 AA standards. This ensures the site is accessible to all users, including those with disabilities.

Overall, these practices demonstrate my understanding and application of Janet regulations, including ethical use of the network, data protection, secure coding practices, copyright compliance, and digital accessibility. By following these principles, the website aligns with both Janet policies and UK legal requirements for web publication.
