Feedback & Program Directory

Technical Description

This page is structured using semantic HTML elements such as <section>, <div>, <form>, and <fieldset> to clearly organise content into meaningful parts like program listings, reviews, and the feedback form. The program directory is designed using a CSS Grid layout (display: grid) with two columns, allowing multiple program cards to appear side by side while maintaining a clean and responsive structure. The overflow-y: auto property enables vertical scrolling within the program section, improving usability when content exceeds the set height.

Each program is presented using reusable card components styled with border-radius, box-shadow, and padding to create a modern and visually appealing layout. Flexbox (display: flex) is used to align icons, headings, and buttons evenly within each card. Buttons such as “Learn more” and “Join” are styled using custom classes with hover effects to enhance interactivity.

The feedback form uses input types like text, email, tel, date, and datetime-local to ensure proper data collection. JavaScript is implemented to provide client-side validation, display error messages, and show a success message upon valid submission. A live character counter is also included for the message field to improve user experience.

Accessibility

Accessibility is considered by using proper form labels linked to input fields, ensuring screen readers can correctly interpret the form. Required fields are clearly indicated, and error messages are displayed to guide users when invalid input is detected. Images include descriptive alt text to support users who rely on screen readers.

The page maintains good colour contrast between text and background for readability, and content is structured logically using headings (<h2>, <h3>) to improve navigation. Interactive elements such as buttons and links are clearly styled and easy to identify.

Team Page

Technical Description

This page is structured using semantic HTML elements such as <section>, <div>, and headings (<h2>, <h3>) to organise content clearly. The team members are displayed using a grid layout created with display: grid and grid-template-columns: repeat(4, 1fr), which ensures that four cards appear evenly across the page.

Each team member is presented using a reusable card component (.team-card) styled with padding, border-radius, and smooth transitions. Images are styled using object-fit: cover to maintain consistent sizing without distortion. A hover effect (transform: scale() and box-shadow) is applied to create an interactive and visually engaging experience.

Additional details about each member are hidden by default using max-height: 0 and opacity: 0. These details become visible on hover or focus using CSS transitions, creating a smooth reveal effect without the need for JavaScript. The hero section at the top uses a background image with background-size: cover and background-position: center to create a visually appealing introduction.

Accessibility

Accessibility is improved by using meaningful alt text for all images, allowing screen readers to describe each team member. The use of tabindex="0" on each card ensures that users can navigate through the team members using a keyboard.

The :focus pseudo-class is used alongside hover effects to reveal additional details, ensuring that keyboard users have the same experience as mouse users. A visible focus outline is also applied to highlight the selected card. Clear heading structure and readable text contrast further enhance usability and accessibility.

Content Page

Technical Description

This page is structured using semantic HTML elements such as <section>, <div>, headings, paragraphs, and lists to organise content into clear thematic sections. Each topic (e.g., causes, consequences, and solutions) is separated into individual content blocks using reusable card-style containers (.card and .stcard) to maintain consistency across the layout.

The layout of each section uses Flexbox (display: flex) to position text and images side by side, improving readability and visual balance. Images are styled with object-fit: cover, border-radius, and box-shadow to maintain consistent sizing and enhance the visual design. A hover effect using transform and transition is applied to images to create a subtle interactive experience.

Internal navigation is implemented at the top of the page using anchor links (<a href="#sectionID">) that allow users to quickly jump to different sections of the page. A “Go to Top” button is also included using a sticky position to improve navigation usability. Lists (<ul>) are used to present actionable steps clearly, and spacing and line-height properties are applied to improve readability.

Accessibility

Accessibility is supported by using meaningful heading structures (<h2>, <h3>, <h4>) to create a logical content hierarchy that can be easily interpreted by screen readers. All images include descriptive alt text to ensure users with visual impairments can understand the content.

Internal navigation links improve accessibility by allowing users to quickly move between sections without excessive scrolling. Text is styled with appropriate line spacing and contrast to enhance readability. Interactive elements such as buttons and links are clearly visible and easy to use, supporting both mouse and keyboard navigation.

Challenges and Lessons Learned

One challenge was maintaining a consistent layout across pages using Grid and Flexbox. This was solved by reusing common styles and adjusting spacing carefully.

Implementing JavaScript for form validation and the character counter was also challenging, but it improved understanding of event handling and DOM manipulation.

Managing image sizes was resolved using object-fit: cover, and accessibility was improved by adding proper alt text and semantic HTML.

Overall, this project improved skills in layout design, interactivity, and maintaining consistency across a website.

Compliance

The pages follow JANET regulations by ensuring all content is appropriate, educational, and does not include harmful or copyrighted material without permission.

Accessibility is considered by using clear structure, readable text, and alternative text for images.

Overall, the website meets guidelines for safe, responsible, and appropriate web publishing. See the lecture notes for more details on compliance.
