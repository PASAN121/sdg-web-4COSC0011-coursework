Home page

Technical Description

The Home Page was developed using semantic HTML5 elements such as <header>, <main>, <section>, and <footer> to ensure a clear and meaningful structure. The page layout is divided into a hero section, a card-based information section, and an interactive gallery..

CSS Grid was used to create responsive layouts, particularly with grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)), allowing content cards to automatically adjust based on screen size. The hero section uses a background image with background-size: cover and background-position: center to maintain visual consistency across different devices.

The gallery section includes hover effects implemented using CSS transitions and transforms such as transform: scale() and translateY(). These effects enhance interactivity and user engagement. The overall design focuses on simplicity, readability, and a modern user interface.

Accessibility

Accessibility was considered by using descriptive alt attributes for all images to support screen readers. Semantic HTML elements improve navigation for assistive technologies. Text contrast was carefully selected to ensure readability, especially in the hero section where white text is placed over a dark overlay.

The layout is fully responsive, ensuring usability across desktops, tablets, and mobile devices. Clear navigation links and structured headings (<h1>, <h2>) also help users understand the content hierarchy.

Gallery Page

Technical Description

The Gallery Page was implemented using structured HTML sections and a responsive grid layout to display images in an organized way. The grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)) property ensures that gallery items automatically adjust based on screen size, providing a flexible and responsive design.

Each gallery item is placed inside a div with the class .cell, containing an image and a label. CSS properties such as object-fit: cover and aspect-ratio maintain consistent image dimensions. Hover effects are created using transform: scale() and box-shadow to enhance interactivity.

JavaScript is used to implement a dynamic modal system. When a gallery image is clicked, event listeners (addEventListener()) capture the interaction and update the modal content. The selected image source, title, and description are dynamically inserted into elements such as modalImage, modalTitle, and modalDescription.

The modalBody element acts as the main container for the popup content. It is dynamically updated using JavaScript to change styles such as background color, font, and text color. Additional controls (dropdown menus) are created using createElement() and inserted into the DOM using appendChild(), allowing users to customize the modal appearance in real-time.

The modal visibility is controlled by toggling the .active class using JavaScript. Smooth animations are achieved using CSS transitions and transforms, creating a modern and interactive user experience without reloading the page.

Accessibility

Accessibility was considered by including descriptive alt attributes for all images, ensuring that users with screen readers can understand the content. Semantic headings (<h2>) are used to clearly structure different sections such as Wildlife Protection, Deforestation, and Take Action.

The modal design ensures readability with proper contrast between text and background. Interactive elements such as buttons and dropdowns are clearly labeled, improving usability. The responsive grid layout also ensures accessibility across different devices including mobile and tablets.

Content Page 01

Technical Description

The Content Page was structured using semantic HTML elements such as <section>, <div>, and <h2> - <h4> headings to organize information into meaningful sections. Each topic, such as wildlife importance, threats, and conservation efforts, is presented using separate sections to improve readability and logical flow.

A flexible layout was created using CSS Flexbox in the .stcard class, allowing text and images to be displayed side-by-side. The display: flex and align-items: center properties ensure proper alignment and spacing between content elements. Images are styled using object-fit: cover and hover effects such as transform: scale() to enhance visual interaction.

Internal navigation is implemented using anchor links (<a href="#section-id">) within the hero section, allowing users to quickly jump to different parts of the page. This improves usability and user experience, especially for long-form content.

Additional styling elements such as the .fact-box are used to highlight important information. CSS properties like border-left, background-color, and border-radius help visually separate key facts from the main content.

A "Go to Top" button is implemented using anchor links, improving navigation and allowing users to easily return to the top of the page without scrolling manually.

Accessibility

Accessibility was addressed by using descriptive alt attributes for all images, ensuring compatibility with screen readers. Proper heading hierarchy (<h2> to <h4>) was maintained to help users understand the structure of the content.

Text readability was improved by using appropriate line spacing, contrast, and font sizes. The layout is responsive and adapts to different screen sizes, ensuring usability on mobile devices, tablets, and desktops.

Internal navigation links and clear section labels improve user interaction and make the content easier to explore, especially for users relying on keyboard navigation.
