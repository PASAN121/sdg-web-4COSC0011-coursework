Splash Screen

Technical Description

I created the splash screen as the first page users see when visiting the NatureGuard website. It includes a welcoming title, a short mission statement, group member names, a loading animation, a countdown timer, and an automatic redirect to the home page. I used HTML to structure the content, embedded CSS to style the full-screen background image and overlay, and JavaScript to create the countdown effect and improve the user experience.

Accessibility

I made the splash screen accessible by using clear readable text, strong colour contrast between the overlay and background image, and meaningful text content. The page also includes a skip button so users can manually continue to the home page without waiting for the automatic redirect.

Impact Simulator

Technical Description

I developed the Action Impact Simulator as an interactive page that allows users to select actions that help protect life on land. I used HTML to structure the action cards, CSS to style the layout and selection states, and JavaScript to let users select and deselect cards, calculate total impact points, display an impact level, and change the background image depending on the result. I also added submit and reset buttons to make the interaction clearer and easier to use.

Accessibility

I improved accessibility on this page by making the cards visually clear and clickable, adding alt text to all images, and ensuring the selected state is clearly visible. I also used readable text, spacing, and buttons that are easy to identify and interact with.

Content Page

Technical Description

I created my individual content page to provide informative content related to the website theme “Life on Land.” I used semantic HTML elements such as headings, paragraphs, sections, and images to organise the information clearly. CSS was used to maintain consistency with the NatureGuard design style and to improve readability and structure across the page.

Accessibility

I made the content page accessible by using clear headings, readable paragraph text, meaningful image alt text, and a clean page structure that helps users easily follow the information. The page layout was designed to remain clear and readable across different screen sizes.

Challenges and Lessons Learned

One of the main challenges I faced was making the Action Impact Simulator both functional and visually consistent with the rest of the website. I had to make sure the JavaScript worked correctly when selecting actions, calculating points, and updating the feedback area. Another challenge was making sure text remained readable when the background image changed dynamically. I solved this by improving layout spacing and using card-style containers to separate important content from the background. Through this process, I learned how to combine HTML, CSS, and JavaScript more effectively to create interactive and user-friendly web pages.

Compliance

The pages I created were designed to follow good web publishing practice by using original or properly credited content, meaningful structure, accessible text, and working internal navigation. I ensured that images included alt text, interactive elements were clearly visible, and the pages were validated to reduce HTML and CSS errors. I also considered readability, usability, and accessibility throughout the design process to make the website suitable for a wide range of users.
